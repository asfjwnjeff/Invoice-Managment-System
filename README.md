# 发票业财中台

这是一个基于 Next.js App Router 的发票与业财一体化原型系统。页面结构参考了现有 HMG 结算平台的收入、成本、代垫、销账、报表模块，也吸收了链票系统在发票池、查验、抬头校验、交付、推送、业务票池和系统配置上的信息架构。

## 功能范围

- 工作台：待开票、未收应收、代垫余额、坏账风险、接口/审计流水。
- 收入管理：开票申请、收入发票查询、未开票收入、已开票收入。
- 发票管理：发票汇总、进项发票池、销项发票池、红冲管理、作废管理、发票交付、链票流水。
- 成本管理：成本发票管理、预付款关联发票、应付匹配。
- 代垫管理：代垫登记、代垫查询、代垫发票查询、超期区域汇总、超期客户汇总。
- 应收销账：应收台账、银行流水维护、银行流水销账、逾期管理、坏账管理。
- 报表分析：项目毛利、发票分析、税务分析、代垫分析。
- 系统设置：企业主体/抬头、客户、供应商、税务规则、业务字段、业务票池、导出模板、系统日志。

## 本地运行

```bash
npm install
npm run dev
```

默认访问：

```text
http://localhost:3000
```

如需指定端口：

```bash
npm run dev -- -p 5173
```

## 验证

```bash
npm test
npm run build
```

当前业务规则测试覆盖：

- 已开票发票自动生成应收。
- 数电票已开具后作废被拦截，并建议红冲。
- 部分红冲保留原票关系。
- 按到期日计算账龄和逾期风险。
- 分批收款核销应收。
- 坏账核销保留应收追溯链。

## Render 部署

仓库内提供 `render.yaml`。当前配置按 Static Site 部署：

- Build Command: `npm install && npm run build`
- Publish Directory: `out`

Next.js 已配置 `output: "export"`，构建后会生成静态站点。
