# 发票业财中台

这是一个基于 Next.js App Router 的发票与业财一体化原型系统。产品口径以链票开票及税务对接为主线，同时参考成熟发票云、税务云和业财税档一体化系统在发票池、查验、抬头校验、红冲作废、交付归档、接口监控、业务票池和系统配置上的通用能力。

## 功能范围

- 经营工作台：经营概览、票财待办、异常中心、链票监控。
- 发票中心：开票申请、开票审核、销项发票、进项采集、进项发票、查验查重、认证抵扣、红冲作废、发票交付、发票归档、链票流水。
- 客户结算：结算单、客户对账、未开票收入、已开票收入、收入发票匹配、收入调整。
- 供应商结算：成本单、供应商对账、成本发票匹配、应付匹配、预付款核销、暂估冲回、成本分摊、成本调整、供应商协同。
- 代垫款管理：代垫登记、客户确认、代垫开票、代垫应收、代垫回款核销、代垫超期、代垫减免/坏账。
- 应收与回款：应收台账、银行流水、收款认领、自动核销、手工核销、逾期管理、催收计划、坏账计提、坏账核销/转回、客户信用。
- 税务合规：税务工作台、销项税额、进项税额、抵扣统计、税负分析、商品编码/税率校验、数电票额度、票源/清卡、申报状态。
- 报表中心：项目毛利、发票分析、开票进度、收票进度、抵扣分析、发票异常、客户开票/回款、供应商成本、应收账龄、应付账龄、暂估余额、代垫分析、税务分析。
- 基础配置：企业主体/开票抬头、客户、供应商、商品/服务编码、税率规则、开票规则、审批流程、单据编号、匹配规则、核销规则、风险规则、业务字段、导出模板、消息模板、链票接口配置、操作日志、接口日志。

## 本地运行

```bash
npm install
npm run dev
```

默认访问：

```text
http://localhost:3000
```

如需指定端口：

```bash
npm run dev -- -p 5173
```

## 验证

```bash
npm test
npm run build
```

当前业务规则测试覆盖：

- 已开票发票自动生成应收。
- 数电票已开具后作废被拦截，并建议红冲。
- 部分红冲保留原票关系。
- 按到期日计算账龄和逾期风险。
- 分批收款核销应收。
- 坏账核销保留应收追溯链。

## Render 部署

仓库内提供 `render.yaml`。当前配置按 Static Site 部署：

- Build Command: `npm install && npm run build`
- Publish Directory: `out`

Next.js 已配置 `output: "export"`，构建后会生成静态站点。

如果在 Render 上误建成 Web Service，Start Command 可使用：

```bash
npm run start
```

该命令会通过 `serve` 托管 `out` 目录，并自动读取 Render 注入的 `PORT`。
